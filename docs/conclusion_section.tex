\section{Conclusion}
\label{sec:conclusion}

This work presented a modular, correctness-first pipeline for \textbf{LLM-based SQL query optimization}. The system integrates prompt construction, candidate generation, SQL normalization, semantic validation, benchmarking, and ranking within a unified evaluation framework.

Using the TPC-DS workload on PostgreSQL, the experiments showed that LLM-generated rewrites can produce meaningful gains on a subset of analytical queries, but the benefits are not uniform across the workload. Among the tested configurations, richer prompt context with direct generation produced the strongest overall result, while the larger-scale evaluation still revealed regressions and invalid candidates on some queries. These results suggest that LLM-based SQL optimization is promising, but remains highly query-dependent and must be supported by strict semantic validation and careful ranking.

One limitation of the current study is that the evaluation was not extended to a broader set of LLM families, which may affect both validity and optimization quality. Future work should therefore compare additional models, refine prompt and ranking strategies, and investigate how query characteristics can be used to predict when LLM-based rewriting is most effective.
