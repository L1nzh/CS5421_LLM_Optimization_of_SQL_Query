### Performance Bottleneck Analysis
The original query runs **15 separate sequential scans on the `store_sales` table** (3 aggregations per bucket × 5 buckets), leading to massive redundant I/O and computation for large datasets. It repeats identical filter conditions for each subquery instead of computing all required aggregates in a single pass.

### Rewrite Steps
1. Compute all bucket-level aggregates (count, average discount, average net paid) in one single scan of `store_sales` filtering only relevant rows with `ss_quantity BETWEEN 1 AND 100`
2. Pivot the aggregated bucket results into the required 5-column output format
3. Join with the `reason` table filter to preserve the original result set semantics (return 0 rows if `r_reason_sk=1` does not exist)

<SQL>
WITH bucket_aggregates AS (
    SELECT
        CASE
            WHEN ss_quantity BETWEEN 1 AND 20 THEN 1
            WHEN ss_quantity BETWEEN 21 AND 40 THEN 2
            WHEN ss_quantity BETWEEN 41 AND 60 THEN 3
            WHEN ss_quantity BETWEEN 61 AND 80 THEN 4
            WHEN ss_quantity BETWEEN 81 AND 100 THEN 5
        END AS bucket_id,
        COUNT(*) AS row_count,
        AVG(ss_ext_discount_amt) AS avg_discount,
        AVG(ss_net_paid) AS avg_net_paid
    FROM store_sales
    WHERE ss_quantity BETWEEN 1 AND 100
    GROUP BY 1
),
pivoted_buckets AS (
    SELECT
        MAX(CASE WHEN bucket_id = 1 THEN CASE WHEN row_count > 62316685 THEN avg_discount ELSE avg_net_paid END END) AS bucket1,
        MAX(CASE WHEN bucket_id = 2 THEN CASE WHEN row_count > 19045798 THEN avg_discount ELSE avg_net_paid END END) AS bucket2,
        MAX(CASE WHEN bucket_id = 3 THEN CASE WHEN row_count > 365541424 THEN avg_discount ELSE avg_net_paid END END) AS bucket3,
        MAX(CASE WHEN bucket_id = 4 THEN CASE WHEN row_count > 216357808 THEN avg_discount ELSE avg_net_paid END END) AS bucket4,
        MAX(CASE WHEN bucket_id = 5 THEN CASE WHEN row_count > 184483884 THEN avg_discount ELSE avg_net_paid END END) AS bucket5
    FROM bucket_aggregates
)
SELECT pb.*
FROM reason r
CROSS JOIN pivoted_buckets pb
WHERE r.r_reason_sk = 1;
</SQL>
